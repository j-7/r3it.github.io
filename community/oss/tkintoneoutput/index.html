kintone Connector for talend / tKintoneOutput

kc4tのTalendコンポーネント tKintoneOutput は、RDBMS等からkintoneへレコードの書き込みを行うコンポーネントです。

インストール方法はこちらをご確認ください。

機能詳細

RDBMSからkintoneへレコードの書き込み

  • スキーマのカラム名をkintoneのフィールドコードとみなし、kintoneにレコードを書き込みます
  • CSVファイルなど、スキーマをサポートするデータソースから、1行毎upsertを実行します

設定パラメータ

Setting for kintone

Parameter Description
API token 接続するkintoneアプリの API token。レコード閲覧・追加・編集権限が必要です。
Sub domain 接続するアプリのサブドメイン
Guest SpaceId もしゲストスペースのアプリケーションの場合、ゲストスペースのIDを指定します
kintone AppId アプリID（数値で指定します）
upsert queries use this Field Code kintoneアプリ上の「プライマリキー」として扱うフィールドコードを指定します。kc4tはこのフィールドコードを使って upsert 処理を行います。

スキーマとキントーンのフィールドのマッピング方法How to mapping schema to kintone’s fields

スキーマのカラムは、キントーンのフィールドと名前でひも付けされます。もしスキーマ側に存在するカラムがキントーン側になかった場合、そのカラムは単純に無視されます（エラーにもなりません）。

コンポーネントの接続方法

tKintoneOutput はRowを受け取ります。 上の例は tFileInputDelimited からRowを受け取っている例です。

実行結果

処理した全てのフィールドは標準出力にダンプされます。もしエラーが発生した場合、エラーを標準出力に吐いてプログラムは停止します。

（この挙動は、エラーステータスを返して正常終了するように修正される予定です。）
