kintone Connector for talend

kintone Connector for talend kc4tは Talend Open Studio 用のkintone接続コンポーネントです。

機能

  • kintoneからJDBC接続可能なRDBMSへのレコードのエクスポート
  • RDBMSからkintoneへレコードのインポート（近日公開予定）

必要な環境

  • kintone SDK for Java version 0.6以降 https://github.com/kintone/java-sdk
  • JDK 7以降

機能詳細

kintoneからJDBC接続可能なRDBMSへのレコードのエクスポート

  • kintoneアプリを元に自動的にテーブルスキーマを作成します
  • レコードを（サブテーブルがある場合は複数の）テーブルにエクスポートします
  • エクスポート時にクエリを指定可能です
  • 実行ジョブのステータスはジョブステータス格納用テーブルに記録されます

インストールと使い方

  • ソースコード/バイナリを含むダウンロードはGitHubリポジトリからお願いします https://github.com/r3it/kc4t

ファイルの設置と設定

  • Talend Open Studioを起動します
  • “カスタムコンポーネント” フォルダを設定します
    • 例として”Custom Components”という名前のフォルダを、ハードディスクのどこかに作成します
    • Studioの設定画面で Window-> Preferences-> Talend-> Components と遷移します
    • “User component folder” にいま作成した”Custom Components”フォルダを設定します
  • ダウンロードしたソースの中のsrcフォルダに有る tKintoneInput フォルダを、あなたの “Custom Components” フォルダにコピーします

設定パラメータ

JDBC設定

Parameter Description
Username JDBC接続時のユーザ名
Password JDBC接続時のパスワード
JDBC URL JDBC URL
Driver Class name JDBCドライバのクラス名

出力先テーブル名の設定

Parameter Description
Export table prefix データをエクスポートするテーブル名の接頭辞
JobReport table name ジョブ結果を格納するテーブル名

kintoneの設定

Parameter Description
API token 接続するkintoneアプリの API token
Sub domain 接続するアプリのサブドメイン
kintone AppId アプリID（数値で指定します）
Guest SpaceId もしゲストスペースのアプリケーションの場合、ゲストスペースのIDを指定します
query クエリパラメータを利用する場合、ここに設定します。何も設定しなければ、コンポーネントはすべてのレコードのエクスポートを試みます。
order by field デフォルトは “RECORD_NUMBER” です。あなたのアプリケーションが “RECORD_NUMBER” を持っていない場合、ソートに利用するフィールド名を指定してください。
retrieve revision field デフォルトは “true” です。チェックを外すと、REVISIONフィールドを取得しません。

出力結果

エクスポートJobが成功した場合、以下の結果パラメータが取得可能です。

Parameter Description
JobId ジョブID（テーブルの接尾詞になります）
MainTable エクスポートされたテーブル名
SubTables サブテーブルがあった場合、それらのテーブル名がカンマ区切りで入ります。

その他

  • 現在のところテーブルは削除されません。ジョブが実行されるたびに、新たにテーブルが作成されます。
