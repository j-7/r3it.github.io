kintone hive Vol.3 大阪に潜入してきました

Posted on 2016-06-10 12:00

Ikegami MIdori

kintoneユーザーによるkintoneユーザーのための大規模ユーザー会！kintone hive とは？

「hive」 は蜂の巣という意味ですが、「活気あふれる場所」という意味合いもあるそうです。

kintone hive（キントーンハイブ）には人とノウハウが集まる活気あふれた場所にしたいという願いがこめられているそう。

2015年5月/10月と、kintone hiveは2回東京で開催されてきました。

そして先日、2016年5月26日(木曜日)に東京でkintone hive Vol.3が開催され、昨日6月9日(木曜日)は 大阪でkintone hiveが初開催されました！

「ユーザー会」 ということで、kintoneを導入することで業務がこんな風に改善された！というようなリアルな事例の紹介の場になっています。

事例発表は、実際に導入に携わったユーザー様ご自身が登壇して発表されるので、まさしく「生の声」が聞けるということですね！

これはkintone導入を検討されているユーザー様には非常にオイシイ內容です(๑´ㅂ`๑)

更に更に！kintoneデベロッパーによるライトニングトークタイム kintone hack もあったりと、kintoneの活用ヒントがふんだんに盛り込こまれた非常に魅力的なコンテンツになっています！

最終的には定員100名を超えたユーザー様がご来場されたようです！皆さんの関心度の高さが窺えますね(p*･ω･)p

ユーザー様のリアルなkintone事例紹介

Vol.3では、全部で4社のユーザー企業様がそれぞれのkintone導入事例について語られていました！

NPO法人スマイススタイル 藤吉様

タイトルは 「仕事をもっと「クリエイティブ」にするシステム導入」

「ハローライフ」という「『働くこと』『仕事』に対する学びの場」で、イベント管理のためにメールワイズとkintoneを連携したシステムを導入されたお話でした。

従来Webの申し込みフォームから届いた参加者情報をExcelにまとめ、メールで事前連絡を行うというすべて手動で行っていた事務作業を、kintoneとメールワイズで連携したシステムを導入することで、事務作業の大幅な削減に成功し、人的ミスも減ったとのこと！ 手動で行っていた事務作業は、イベントが開催されるたびに同じ作業を繰り返すので、月に20回以上も同じ作業を行っていたようです。

ここを自動化できたので、相当な時間短縮につながったとのこと！

またサイボウズさんの導入相談Cafeで相談をされて、運用開始までわずか1ヶ月という短期間で稼働できるようになったのも、kintoneというファストシステムならではといった印象でした₍₍ (ง ˘ω˘ )ว ⁾⁾　

そして、当時実はこの導入相談Cafeで相談を受けたのが弊社浅賀だったとのこと！

世間はほんとに狭いなと思った瞬間でした。

また、導入後に苦労した点も少なからずあったようです。

操作が簡単というのは、パソコンやインターネットに慣れ親しんだ世代の観点です。

オフィスにはいろんな世代の方が働いているため、ご年配の方には操作方法等わかりやすい言葉で説明する必要があったとのこと。

わたしたちが対面開発をさせて頂く時においても、非常に参考になる意見でした。

今後はイベントの管理だけでなく、どんどん活用範囲を拡大される予定らしいです。

また、現場からも「こういったことがkintoneでできるのではないか」という前向きな声がたくさん上がってきているようですよ！

株式会社トーア紡コーポレーション 中井様

タイトルは 「kintoneで変わる」

情シス部門の観点からのお話です。

ずっと内製でシステム開発をされてきた中で、ある時24/365に耐えられる社内システムの構築が必要となったそうです。

そんな中で出会ったのがkintone。

kintoneにすることで、開発のコストが大幅に削減される。 バグもなく、画面設計も容易、帳票の作成も出来、環境にも依存しない、外部システム連携も出来てしまうといった点で、それまですべて自前で開発してきた中井様は「してやられた感」を感じたそう。

外部連携やカスタマイズを考えると、kintoneのライセンスはどうしてもスタンダードコースが必須になってしまうので、一般的にはやはり高価なイメージってありますよね。

しかし運用/保守コストや可用性を考えると安価！確かにそうです。

kintoneを使うと開発してテストまでのフェーズも時間がかからず簡単です。

そのメリットを存分に活かしてkintoneで出来る可能性をいろいろ模索されているみたいですよ。

神戸デジタル・ラボ 武富様

タイトルは「人類 kintone マスター化計画」

壮大なタイトルで始まった、kintone の導入から社内への浸透までのお話しでした。

部署横断で使われる社内システム「todos」の開発にあたり、コスト削減のために導入したはずの自動生成ツールに問題が多く、改善担当を任された武富さん。

ノンプログラミング、サポート体制、クラウド、スマホ対応といったいくつかの条件にマッチしたのが kintone でした。

とは言え、利用する社員から不安の声が挙がる事はシステムのリプレイスではよくある事。 武富さんはスムーズな導入のために、いくつかの取り組みをされました。

  • 事業部の代表者で構成されたワーキンググループでの検討
  • モニター利用者を募集してプレリリース
  • 利用者向け説明会の開催
  • kintone API を使ったデータ移行
  • 積極的にコミュニティ（kintone Café等）に参加して情報収集

これらの取り組みにより、新「todos」が完成したわけです。

武富さんの野望はこれだけにはとどまらず、社内で kintone マスター（kintone のアプリが作れる人）を増やしたいという強い思いから、他部署も巻き込んで kintone マスターをどんどん増やしているそうです。

コツは「近いところからファンを作る」ことだそう。

今回の登壇を機に、社長も巻き込んでいきたいという熱い想いもあるそうです！

株式会社日阪製作所 佐々江様

タイトルは「業務を劇的に変える」

日阪製作所様は、かなり古くからkintoneを利用されているお客様。

事業部統合後もシステム統合がなかなか出来ず、業務もそのまま従来のやり方を継続してしまっていた実状があった様子。

そんな中kintoneと出会い、社内でkintoneプロジェクトが立ち上がったとのこと。

この立ち上げメンバーの中には、現場で働く非エンジニアの女性の方がおり、そういった現場の女性達の要望を、kintoneで形にしていった様子・変遷をお話されていました。

最初はやはりkintoneの導入自体には反対の声もあったようですが、地道な取り組みで徐々に社内の中での評価も得ていったようです。

今ではアプリも営業系・製造系・サービス系、品質保証系、BI系と非常に多岐に渡る種類のアプリを運用されています。

kintoneを利用することで従来バラバラになっていた情報の一元化に成功し、現在もプロジェクト拡大中らしいですよ！
