kintone Café 大阪 Vol.3で実施したカスタマイズハンズオンの概要について(前編)

Posted on 2015-07-21 14:04

Ikegami MIdori

2015年7月15日水曜日に、kintone Café 大阪 Vol.3でJavaScriptを利用したカスタマイズハンズオンを行いました。

今回はその時のハンズオンの内容を少し補足した形でブログを書こうと思います。

わたしのこと

kintone Café大阪には前職の時から参加はしていたものの、早々と登壇デビューを飾ることになってしまったいけがみみどりです。

まだまだサンプルコードに毛が生えた程度のスクリプトしか書けないわたしが、公開処刑のごとくカスタマイズハンズオンに挑戦することに！

ということで、kintone Café大阪で行ってきたカスタマイズハンズオンの概要について、ざっくりブログでご報告したいと思います( ˘ ³˘)❤ ﾟ+｡:.ﾟ

そもそもカスタマイズってどんなこと？

kintoneにおけるカスタマイズは、ざっくり説明すると以下のようなことを指します。

  • データの取り込み、外部システムとの連携
  • JavaScript/CSSを利用して高度な機能を利用する

要は画面上で行える設定以上のことをするイメージですね(‘ω’*)

もちろん、標準機能だけでもそれなりに見た目の良いアプリがノンプログラミングで簡単に作れてしまいます。

Excelやcsv形式のデータであればそのままの形式を維持してインポート出来たりもするので、既存の管理方法から大きな変更を加えることなくクラウドにデータを移行することも可能になります。

しかし今回のkintone Caféは少し応用編！ …ということで、今回は参加者の皆様に、JavaScriptを利用したカスタマイズに挑戦して頂きました。

用意したアプリについて

今回そんな参加者の方々にご用意させて頂いたのは、当方で予め作成したJavaScriptのサンプルコード。

以下の2つです。

  • バリデーション
  • 住所から地図を表示するアプリ（OpenLayerを利用）

このサンプルコードを、用意した顧客リストアプリに登録して挙動を確認します。

顧客リストは、一般的な名前や住所等を登録することが出来るものです。


サンプルコードの説明に入る前に、まずはkintoneにおけるJavaScriptのカスタマイズのルールについて簡単におさらいしましょう。

JavaScriptの記述ルール

  • 拡張子は.jsで保存
  • ファイル名の拡張子は表示しておいたほうが良い。 （じゃないとテキストとかで保存されててスクリプト動かないよ！なんてことも…）
  • 文字コードはUTF-8で保存 （異なる文字コードで保存すると文字化けする原因に！）
  • 即時関数パターンで記述
  • strictモードを使用

即時関数パターンとは

  • 関数を定義するのと同時に実行もする。
  • スコープ汚染防止（kintoneで即時関数を採用している一番の理由）

スコープ汚染についての説明は以下のサイトに詳しく書かれています。

http://analogic.jp/immediate-function/#scope

記述方法 

(function () {
    // 処理の内容を記述する
}());

strictモードとは

  • “use strict”と宣言する
  • 記述ミス防止目的
  • 明示的に「使う」と宣言しないと使えないモード

記述方法 

(function () {
　　”use strict”;
    // 処理の内容を記述する
}());

kintone独自のイベント処理について

kintoneにはkintone独自のイベントがあります。

https://cybozudev.zendesk.com/hc/ja/sections/200418540

  • レコード一覧イベント（表示、保存、変更など）
  • レコード表示イベント（詳細画面表示、削除前など）
  • レコード追加イベント（表示、保存など）
  • レコード編集イベント（表示、保存など）
  • グラフ表示イベント （PC用のみ）

どのタイミングで「処理を実行するか」ということを宣言するものです。

例えば…

レコード追加時、保存するタイミングで処理を実行させたい場合は、下記のような書き方になります。

ex.新規レコード追加時、保存するタイミングで処理Aを実行する。

(function () {
　　"use strict";
　　// 新規レコード追加時のイベント
    kintone.events.on('app.record.create.submit', function (event){

    // （処理A）
    ・・・
}());

ここでは、複数イベント指定することも可能です。

ex.新規レコード追加時または既存レコード編集時、保存するタイミングで処理Aを実行する。

(function () {
　　"use strict";
　　// 新規レコード追加時または既存レコード編集時のイベント
    kintone.events.on(['app.record.create.submit', 'app.record.edit.submit'], function (event){

    // （処理A）
    ・・・
}());

以上が、kintoneにおける基本的なJavaScriptによるカスタマイズのルールになります。
