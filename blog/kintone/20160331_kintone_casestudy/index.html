kintone & メールワイズで効率アップ！！

Posted on 2016-03-31

Koji Asaga

今回はいつもと趣向を変えて、kintone とメールワイズを組み合わせた事例をご紹介してみたいと思います。

メールワイズって？

メールワイズをご存じ無い方もいらっしゃると思うので、まずは簡単に製品のご紹介から。

メールワイズは kintone と同じく、サイボウズさんが提供されているクラウドサービスです。（パッケージ版もあります）

メールワイズのマニュアルには以下のように書かれており、問い合わせ窓口や部署用アドレスのように、複数人で同じメールアカウントを使っている時にとても便利に使う事ができます。

メールワイズとは、企業の代表メールや問い合わせメールを一元管理するためのシステムです。 複数人での作業でも、Webブラウザーを経由して同じメールを操作できるため、メール対応業務の効率化を実現します。

R3社内のメールワイズ活用

R3 は kintone を更に便利にするサービス gusuku を展開しています。

お客様からの問い合わせ対応にメールワイズを使っており、たくさんのご意見・ご要望がメールワイズに蓄積されていっています。

もちろん、gusuku の機能改善の参考にさせて頂いています！


これはこれで非常に便利なのですが、メールワイズはあくまでメールの管理を行うツール・・・。
情報をストックする事はあまり得意ではなく、細かい集計や要望の見える化ができていませんでした。


集計？見える化？と言えば、そう kintone しかありませんよね。

というわけで、メールワイズの情報を kintone に取り込める仕組みをハイスピードに作ってみました (∩´∀｀)∩ﾜｰｲ

できあがったもの

実は kintone と同様に、メールワイズにも JavaScript を読み込む機能があります。（クラウド版のみ）

JavaScript カスタマイズ

今回はこちらの機能を使い、メールワイズで受信したメールの内容を kintone アプリに保存できるようにしてみました。


メールワイズに【kintoneへ登録ボタン】が！


ワンクリックで kintone に蓄積されるようになっています。
（メールの文章全てでは無く、選択された部分だけを kintone に登録する事もできるようにしました）

ソース大公開

メールワイズのカスタマイズは公式ドキュメントが無いため、画面をカスタマイズする際は自分で DOM 構造を解析する必要があります。 メールワイズに関係する部分を抜粋して公開させて頂きます。

メール本文の取得

var mail_plain = $("tt");	//プレーンテキストの場合
var mail_html = $("#mail-body-area");	//HTMLメールの場合
if(mail_plain.length > 0){
	mail = mail_plain[0].innerText;
}
else if(mail_html.length > 0){
	mail = mail_html[0].innerText;
}

日付の取得

var mwdate = $(".mailContentDateTime");
if(mwdate.length > 0){
	var mwdate_text = mwdate[0].innerText;	//日付を取得
	var mwdate_date = mwdate_text.split("\n")[0];	//日付<改行>曜日・時間から日付部分を抽出
	maildate = mwdate_date.replace(/\//g,"-");	//kintoneのフォーマットに変更
}

おわり

今回は kintone とメールワイズの組み合わせでしたが、他にも便利なサービスがたくさんあります。

これからもクラウドサービスを使った事例をご紹介していきたいと思いますので、楽しみにしていてください！
