ハイブリッドクラウドのこれから

Posted on 2014-11-11 09:36

Toshiyuki Konparu

こんにちは、金春です。

クラウドファーストが当たり前の時代になりつつありますが、今後大きな流れになりそうなハイブリッドクラウド。 弊社の考えるハイブリッドクラウドについてお話したいと思います。

ハイブリッドクラウドって？

現在、ハイブリッドクラウドと言えばこんな感じです。

既存のデータセンターとクラウド（ここではAWS）を専用線で接続（AWS Direct Connect）あるいはインターネット経由でVPNで接続してシームレースに利用するパターンですね。

これにより、既存資産とクラウド上に構築した資産を柔軟に連携させて利用することができます。

企業によっては情報の種類によってはクラウドに置けないルールになっているというような場合でも、データに合わせて置き場所が違うというだけで、クラウドに置けるものはクラウドに置くことによってIT投資を最適化することができます。

例えば、

  • 顧客情報はデータセンターの置く
  • 重い分析処理はクラウドを利用して効率よく計算する

というようなことができるようになります。

これからのハイブリッドクラウド

さて、上のハイブリッドクラウドもとても良いものですが、これからのハイブリッドクラウドは少し違った形になると考えています。

上の図と比べると要素が増えています。 増えたのは「Other Cloud Service」と書いている他のクラウドサービスです。 弊社ではここに、kintoneを利用することをお勧めしています。

上で、「情報の種類によってはクラウドに置けないルール」というお話をしましたが、これは今後変わってくると考えています。 まだなかなか認知されていない感じはしますが、クラウドのセキュリティは一般的に企業が独自に構築するデータセンターより高いものになっています。

想像してみてください。

  • 御社にセキュリティ専門のエンジニアはいますか？
  • セキュリティ対策のためにどれだけのコストをかけられますか？
  • 御社に銀行にお金を預けるより高いセキュリティで現金を保管できますか？（それがデータでも同じです）

クラウドのセキュリティの高さは徐々に認知されつつあります。

そうなると、左にある既存のデータセンターが担う役割はどんどん小さくなると考えられます。

その次に起こるのは、単一のクラウドではできないことを他のクラウドを利用して補完しようという動きです。

弊社では、kintoneとAWSを連動させたシステム開発を推奨しています。

kintoneを用いて高速にシステム開発を行い、kintoneにはできないことをAWSで補完する。 まだ残っている既存データセンターについてもAWSを介してkintoneと連携させることで既存資産も有効に活用する。

これからのハイブリッドクラウドは、「クラウドとクラウドのハイブリッド」になると考えています。

まとめ

これからのハイブリッドクラウドは「クラウドとクラウドのハイブリッド」になると考えています。 しかし、すべてのクラウドサービスを完全に把握してシステムを構築することは容易ではなく、 また、あまり多くのクラウドサービスを統合しすぎてもシステムが複雑化するだけになってしまいます。 弊社では、AWS、kintoneを駆使するシステム開発をおすすめしています。

両者のパートナーである弊社にしかできない、今後のサービス展開にご期待ください。
